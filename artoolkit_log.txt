    - Added simpleLiteWinRT, including onscreen controls for tracking parameters.
    - Added multiCubeWinRT, including a method which allows for rendering of "masking" objects via stenciling.
    - Added nftSimpleWinRT using the gibraltar pattern
    - Added uiControlsWinRT, a WinRT pure XAML sample which is not using D3D at all but WriteableBitmap to show the video frame and using Matrix3d to apply the AR transformations to XAML controls.

    - ARToolKitComponent.component and ARgsubD3D.component debug builds are not built with the "d" suffix (as other DLLs are) because a WinMD files must have the same name as their root namespace. Instead we place the debug variant into a subdirectory of "bin-***" named "Debug".
    - Removed "ansicstr" versions of the string conversion functions due to variable scope issues, and instead use "ansis" variant with ".cstr()" at each location this is required.
    - For WinRT, strdup() is defind to _strdup(), since the former results in spurious runtime exceptions when running in debug mode.

    Major release 5.3.
    
    - New libKPM with FREAK detector replaces SURF. libKPM license is now LGPLv3.
    - libKPM no longer depends on OpenCV's FLANN implementation (libopencv-flann), so NFT examples no longer need to link against it either. calib_camera still depends on OpenCV.
    - Retain C interface to libKPM, kpmHandle is now opaque type with definition in kpmPrivate.h. Added some additional accessors. kpmProcMode is now an enum.
    - Change references to "fset2" to "fset3" in utilities and all examples.
    - Reworked some Eigen headers to avoid warnings around initialisation order.
    - Switched all platforms from libstdc++ to libc++ with C++11 (i.e. C++0x).
    - Bundled OpenSceneGraph on iOS and Android is now v3.2.2RC3, linked against libc++.
    - OS X build builds against OpenSceneGraph v3.2.2RC3, linked against libc++, supplied separately.
    - Android SDK now builds with Clang toolchain and links against libc++_shared. libc++_shared must be loaded in Java code for projects that need it, including those using libKPM, libARWrapper, OpenCV, or OpenSceneGraph.
    - While ARToolKit for Android is 64-bit ready, the SDK remains 32-bit for this release due to lack of 64-bit support in some prerequisites.
    - For some reason, some Android platforms define GL_BGRA_EXT but not GL_BGRA, so manually define in this case.
    - iOS OSG apps must now link against MobileCoreServices.framework.
    - iOS bundles now show app name correctly in dialogs (set CFBundleDisplayName).
    - Added video library, build and packaging files, and a draft C++/CX component for WinRT-based platforms, including Windows Phone 8.1 and Windows Store 8.1.
    - Android: Update libjpeg library from https://github.com/android/platform_external_jpeg.git. It still needs 'extern "C"' wrappers from c++ though.
    - Android: Supply splash screen in correct format for calib_optical.
    - Android: Allow build.sh and build_native_examples.sh to be called with second parameter 'clean'.
    - Coverity ART high impact issue resolutions.
    - Added some missed header inclusions.
